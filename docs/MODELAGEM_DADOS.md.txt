# 🗄️ MODELAGEM DE DADOS — Donnexa

> Última atualização: Julho de 2026

---

# Objetivo

Este documento descreve a estrutura do banco de dados do Donnexa.

A modelagem foi desenvolvida seguindo princípios de normalização, visando facilitar a manutenção dos dados, reduzir redundâncias e permitir a evolução do sistema para SQL Server e futuras aplicações web e mobile.

---

# Arquitetura

O Donnexa utiliza atualmente uma estrutura relacional composta por tabelas independentes conectadas através de chaves.

O modelo foi pensado para crescer sem necessidade de grandes alterações estruturais.

Atualmente os dados são armazenados em Excel e consumidos pelo Power BI.

Futuramente serão migrados para SQL Server.

# Decisões de Arquitetura

Durante a modelagem algumas decisões importantes foram tomadas.

## Separação entre Livro e Leitura

Um livro representa uma obra.

Uma leitura representa um evento.

Isso permite registrar releituras sem duplicar livros.

---

## Separação entre Livro e Biblioteca

Nem todo livro cadastrado pertence à biblioteca.

Também é possível desejar adquirir um livro sem possuí-lo.

Essa decisão permitiu criar funcionalidades como:

- Lista de desejos
- Controle de posse
- Futuro controle de empréstimos

---

## Uso de Tabelas de Dimensão

Autores, editoras, gêneros e formatos foram separados em tabelas próprias.

Benefícios:

- Evita repetição de dados.
- Facilita filtros.
- Melhora a modelagem.
- Aproxima o projeto de um Data Warehouse.

# Considerações Finais

A modelagem do Donnexa foi construída com foco em escalabilidade, organização e reutilização dos dados.

Embora atualmente utilize Excel como fonte de dados, toda a estrutura foi pensada para suportar futuras evoluções sem necessidade de remodelagem significativa.

O objetivo é que o banco de dados acompanhe a evolução do projeto, tornando-se a base para uma plataforma completa de gestão de leitura.
